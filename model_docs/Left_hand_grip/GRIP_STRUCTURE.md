# GRIP_STRUCTURE

`グリップ構造` の考え方と開発方針をまとめる README です。

この構造は `内殻` や `外殻` と違い、最初から厚みを持った閉じたボディとして作る前提ではありません。まずは点・線・面で構成された、一枚の折り紙のような展開構造として定義し、その後に `外殻基本構造` と隙間なく接続させることで、はじめて質量を持つ 3D プリント可能なボディとして成立させます。

補足:
この README では説明上 `外殻基本構造` という呼び方を使います。現行コード上の名称は `外殻基準構造` です。

## 構造の考え方

- `グリップ構造` は、点・線・面の関係で形を決める構造です。
- 初期段階では、厚みを持つソリッドというより、折り紙のような一枚構成のサーフェス的な扱いを前提にします。
- この一枚構成のエンド部分を `外殻基本構造` に隙間なく接続し、閉じた境界を成立させることで、最終的に質量を持つボディへ発展させます。
- したがって `グリップ構造` 単体の形状定義と、`外殻基本構造` への接続条件は、同じくらい重要な仕様として扱います。

## 基本方針

- `グリップ構造` の形状は、面の寄せ集めではなく、名前付きの点・線・面の関係として管理します。
- 各点・線・面には一意な名称を付けます。
- 寸法や角度は固定値として埋め込まず、外部入力で与えたり差し替えたりできる前提で管理します。
- `内殻` や `外殻基本構造` との位置関係も、外部から与える `X` `Y` `Z` 情報または参照情報として扱います。
- 将来的に形状調整が発生しても、名称付き要素と入力値を差し替えることで構造全体を再生成できる状態を目指します。

## 要素の管理方針

### 点

- 主要点には名前を付けます。
- 各点は、絶対座標で固定するのではなく、基準点からの距離、角度、他点との関係で表せるようにします。
- 必要に応じて、`内殻` や `外殻基本構造` 上の参照点に従属させます。

### 線

- 線には始点・終点・接続先面との関係が分かる名前を付けます。
- 長さは外部入力値で変更できる前提にします。
- 線の角度も外部入力値または基準面に対する角度として管理します。

### 面

- 面には境界を構成する点・線が追える名前を付けます。
- 面は単独で閉じたソリッドを作るためではなく、折り曲げ前提の一枚構造を定義する単位として扱います。
- どの面が `外殻基本構造` への接続終端になるかを明示します。

## 外部入力として扱う情報

- 長さ
- 角度
- `X` `Y` `Z` 方向の位置情報
- 他ボディとの相対位置
- 接続先となる点・線・面の参照情報

これらは今後、`grip.py` 側でまとめて受け取れる形に整理していきます。

## 外殻基本構造との接続方針

- `グリップ構造` のエンド部分は、`外殻基本構造` と隙間なく接続する前提です。
- 接続がずれると、最終的に閉じたボディにならず、3D プリント可能な形状として成立しません。
- そのため、接続終端にある点・線・面は通常部より優先度の高い管理対象として扱います。
- 将来的には、接続面の一致、端点位置の一致、法線方向の整合を確認できるようにします。

## 現時点の実装方針

- `3.py` は `グリップ構造` 単体確認用の入口として使います。
- `fusion_app/grip_structure.py` は、必要な段階になれば内部的に `内殻` と `外殻基準構造` を参照生成してから `grip.py` を呼べる構成にします。
- ただし現時点の `3.py` は軽量な単体確認を優先し、参照ボディをまだ生成しません。`グリップ構造` だけを表示対象にします。
- `グリップ構造` の形状処理本体は `fusion_app/grip.py` の `build_grip()` に集約します。
- 現在はテスト用の仮面も削除してあり、`grip.py` はまだ本実装前の空の入口です。

## 実装時の最初の整理単位

1. 名前付きの基準点一覧を決める
2. その点を結ぶ基準線一覧を決める
3. 基準線から構成される面一覧を決める
4. `外殻基本構造` に接続する終端要素を特定する
5. 各要素に対して、長さ・角度・位置のどれを外部入力にするか決める
6. その情報を `grip.py` で再生成可能な形に落とし込む

## 確認方法

1. `fusion_app/grip.py` の `build_grip()` に `グリップ構造` の処理を追加します。
2. 単体確認は `3.py` から行います。
3. `内殻 + 外殻基準構造 + グリップ構造` の位置関係確認は `1.py` または `2.py` から全体組み立てを実行します。
4. 現時点では `grip.py` が未実装のため、`3.py` 実行時に表示される `グリップ構造` ボディはありません。
